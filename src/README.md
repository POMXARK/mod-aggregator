# Frontend (Svelte 5)

Frontend часть приложения на Svelte 5 с TypeScript.

## Структура

### components/
Основные компоненты приложения:

- **App.svelte** - Главный компонент приложения, управляет роутингом между страницами
- **Sidebar.svelte** - Боковая панель навигации с возможностью скрытия
- **ParserBuilder.svelte** - Визуальный конструктор парсеров с графом нод
- **PageViewer.svelte** - Компонент для просмотра и сохранения веб-страниц
- **ModsList.svelte** - Список модов с карточками
- **SitesManager.svelte** - Управление сайтами (добавление, редактирование, удаление)
- **NotificationsPanel.svelte** - Панель уведомлений о новых версиях модов

### components/nodes/
Компоненты нод для конструктора парсеров:

- **SelectorNode.svelte** - Нода выбора элементов
- **ExtractNode.svelte** - Нода извлечения данных
- **FilterNode.svelte** - Нода фильтрации
- **TransformNode.svelte** - Нода трансформации данных
- **OutputNode.svelte** - Нода вывода результата

### lib/
Утилиты и composables:

- **tauri-wrapper.ts** - Обертка для вызова Tauri команд
- **tauri-mock.ts** - Mock для работы в браузере
- **api.ts** - API интерфейсы
- **composables/** - Переиспользуемая логика:
  - **useSites.ts** - Работа с сайтами
  - **useMods.ts** - Работа с модами
  - **useNotifications.ts** - Работа с уведомлениями
  - **useUI.ts** - Управление UI состоянием

## Технологии

- **Svelte 5** - Фреймворк с runes ($state, $derived, $effect)
- **TypeScript** - Типизация
- **@xyflow/svelte** - Граф нод для конструктора
- **Tailwind CSS** - Утилитарные стили

## Стилизация

- Используются относительные размеры (vw, vh, rem, clamp)
- Темная тема (#0f172a, #1e293b, #334155)
- Адаптивный дизайн
- Плавные анимации и переходы
