# Настройка Cursor AI для проекта

Этот документ описывает все настройки Cursor AI для лучшей работы с проектом.

## Созданные файлы конфигурации

### 1. `.cursorrules` (основной файл)
Главный файл с правилами для Cursor AI. Содержит:
- Инструкции по предложению множественных вариантов улучшения
- Проактивные предложения (как ChatGPT)
- Примеры правильного поведения
- Технические детали проекта

### 2. `.cursor/rules/` (структурированные правила)
Директория с правилами, организованными по категориям:

- **frontend.mdc** - правила для Svelte 5 компонентов
- **backend.mdc** - правила для Rust/Tauri backend
- **code-patterns.mdc** - общие паттерны кода

### 3. `PROJECT_CONTEXT.md`
Подробное описание проекта для AI:
- Архитектура проекта
- Ключевые компоненты
- Модели данных
- Потоки данных
- API между frontend и backend

## Как это работает

### Автоматическое применение правил
Cursor AI автоматически применяет правила из `.cursor/rules/` в зависимости от:
- Открытых файлов (по glob паттернам в frontmatter)
- Контекста задачи
- Типа файлов

### Основные правила из `.cursorrules`
- Всегда предлагать минимум 2-3 варианта решения
- Быть проактивным и предлагать дополнительные улучшения
- Использовать конкретные действия вместо общих советов
- Задавать вопросы для уточнения предпочтений

## Дополнительные улучшения

### 1. Комментарии в коде
Добавляй JSDoc комментарии к функциям для лучшего понимания контекста:

```typescript
/**
 * Загружает сайт в PageViewer и создает ноды из конфигурации парсера
 * @param site - объект сайта с конфигурацией парсера
 */
async function handleLoadSite(site: Site) {
  // ...
}
```

### 2. README файлы в модулях
Создавай README.md в важных директориях для описания их назначения.

### 3. Типизация
Всегда используй TypeScript типы - это помогает AI лучше понимать структуру данных.

### 4. Консистентное именование
Следуй соглашениям об именовании из `code-patterns.mdc`.

## Проверка работы

После создания этих файлов Cursor AI должен:
1. ✅ Предлагать несколько вариантов решения задач
2. ✅ Понимать архитектуру проекта
3. ✅ Следовать паттернам кода проекта
4. ✅ Предлагать проактивные улучшения
5. ✅ Задавать уточняющие вопросы

## Обновление правил

Если нужно обновить правила:
1. Отредактируй соответствующий файл
2. Cursor AI автоматически подхватит изменения
3. Можешь перезапустить Cursor для применения изменений

## Дополнительные рекомендации

### Для лучшего понимания контекста:
- Держи открытыми ключевые файлы проекта
- Используй комментарии в коде для сложной логики
- Обновляй `PROJECT_CONTEXT.md` при значительных изменениях архитектуры

### Для лучшего запоминания:
- Используй консистентные паттерны кода
- Следуй соглашениям об именовании
- Документируй важные решения в комментариях
